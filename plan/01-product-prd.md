# 01. 产品定位与需求分析

## 1. 项目定位

项目名称：`智能行业情报分析师`

一句话定义：
面向打工人的行业资讯效率工具，自动抓取目标行业网站的新闻、公告、通知与数据表格，完成清洗、结构化、向量化和问答分析，并输出可执行的日报。

## 2. 目标用户

- 行业研究员：需要快速扫行业新闻、公司公告、政策变化
- 产品经理 / 战略分析：需要判断行业趋势与竞品动态
- 销售 / 商务：需要拿到客户行业热点、政策利好和风险提示
- 运营 / 投研 / 咨询实习生：需要节省“搬运信息 + 手工整理日报”的时间

## 3. 核心痛点

- 信息分散在多个网站，人工刷新成本高
- 公告页经常混有杂乱 HTML 表格，手工整理很慢
- 看完一堆文章后，难以快速回答“今天最重要的 3 件事是什么”
- 资讯搜得到，但无法沉淀为可追问、可复用、可汇报的知识库

## 4. MVP 场景

以一个行业站点为起点，完成以下闭环：

1. 抓取新闻列表页 / 公告页
2. 如有登录要求，支持 `requests` 登录或 `Selenium` 模拟登录
3. 获取文章正文、发布时间、标签、附带表格
4. 用 `Pandas` 抽取 HTML 表格，导出结构化数据
5. 把文章内容和表格摘要切片后写入向量库
6. 用户通过 API 发起查询，系统基于 RAG 回答并给出来源
7. 系统按天生成“行业日报”

## 5. 为什么不是普通个人知识库

这个项目必须体现“效率工具”属性，而不是简单文件问答。关键差异有：

- 数据源来自外部行业站点，而不是用户手动上传
- 强调资讯时效性和增量更新
- 输出的是“结论 + 风险 + 建议动作”，而不是只返回原文摘要
- 提供日报能力，符合真实工作流
- 通过 MCP 暴露给外部 Agent，实现跨工具协作

## 6. MVP 功能清单

### P0 必做

- 站点配置驱动的新闻抓取器
- requests / Selenium 登录抓取
- Pandas HTML 表格结构化导出
- 文章 JSON 持久化
- Chroma 向量库写入与检索
- LangChain 问答 API
- 日报生成 API
- MCP 工具封装

### P1 建议补充

- 多站点增量抓取
- 关键词监控 / 订阅
- 定时任务与日报自动推送
- 查询历史与用户会话
- 简单前端工作台

### P2 面试加分

- 结果重排与引用高亮
- 多日报模板（销售版 / 战略版 / 产品版）
- 告警机制（关键词突增、政策预警）
- Docker 化部署与 CI

## 7. 成功标准

- 能抓到一个真实行业站点的数据并落盘
- 能从资讯中抽出结构化表格并导出 CSV
- 能通过 API 进行基于来源的问答
- 能生成带“重点、影响、行动建议”的日报
- 能用 MCP 让外部 Agent 调用查询 / 报告能力
